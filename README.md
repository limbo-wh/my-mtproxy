# my-mtproxy

Деплой MTProto-прокси с FakeTLS одной командой (Caddy + alexbers).
Меню на TUI: установка, безопасность, управление, удаление.

## Быстрый деплой

```bash
# На свежем VPS (Ubuntu 22.04+ / Debian 12, под root):
git clone git@github.com:Idol95444/my-mtproxy.git
cd my-mtproxy
sudo bash manage.sh
```

Откроется меню:

```
1. Установить прокси            ← деплой (домен, секрет, AD_TAG)
2. Настроить безопасность VPS   ← ufw, fail2ban, автообновления, sysctl
3. Управление прокси            ← логи, статус, рестарт
4. Обновить скрипт из git       ← git pull последней версии
5. Удалить всё                  ← снести контейнеры и конфиги
6. Выход
```

## Что делает «Установить прокси»

1. Спрашивает **DOMAIN**, **BASE_SECRET** (или генерирует), **AD_TAG** (опц.)
2. Проверяет DNS
3. Устанавливает Docker и docker-compose если их нет
4. Клонирует [alexbers/mtprotoproxy](https://github.com/alexbers/mtprotoproxy) (stable)
5. Генерирует `Caddyfile` и `config.py` из шаблонов
6. Запускает Caddy (порты 80/443, получает LE-сертификат)
7. Запускает alexbers (порт 853, FakeTLS)
8. Печатает FakeTLS-ссылку и чек-лист оставшихся шагов в @MTProxybot

## Что делает «Настроить безопасность VPS»

1. **Сканирует открытые порты** через `ss -tln`
2. Авто-определяет порт SSH из `/etc/ssh/sshd_config`
3. **Про каждый незнакомый порт спрашивает**: оставить открытым или закрыть
4. Настраивает **ufw**: открыты SSH-порт, 80, 443, 853 + одобренные пользователем
5. Ставит **fail2ban**: бан IP на 1 час после 3 неудачных SSH-попыток
6. Включает **автообновления** security-патчей (рестарт в 04:00 если нужно)
7. Применяет **sysctl-защиту**: от спуфинга, SYN-flood, MITM, Smurf

## Архитектура

```
Интернет -> :80  -> Caddy (автопродление LE-сертификата, редирект HTTP→HTTPS)
         -> :443 -> Caddy (TLS, страница-заглушка "OK", маскировка под nginx)
         -> :853 -> alexbers (FakeTLS, забирает сертификат с Caddy при старте)
```
