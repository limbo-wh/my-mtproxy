# my-mtproxy

Деплой MTProto-прокси с FakeTLS одной командой (Caddy + alexbers).
Меню в стиле control panel с живым статусом контейнеров и файрвола.

## Быстрый деплой

```bash
# На свежем VPS (Ubuntu 22.04+ / Debian 12, под root):
git clone https://github.com/limbo-wh/my-mtproxy.git
cd my-mtproxy
sudo bash manage.sh
```

После первого запуска появится команда `proxy` — дальше можно из любого места:
```bash
sudo proxy
```

Откроется меню:

## Что делает «Установить прокси»

Wizard в 3 шага: домен → секрет (или авто-генерация) → AD_TAG.
Затем DNS-проверка, сводка перед установкой, пошаговый прогресс с ✓ для каждого шага.

1. Устанавливает Docker и docker-compose если нет
2. Клонирует [alexbers/mtprotoproxy](https://github.com/alexbers/mtprotoproxy) (stable)
3. Генерирует `Caddyfile` и `config.py` из шаблонов
4. Запускает Caddy и ждёт LE-сертификат
5. Запускает alexbers
6. Печатает FakeTLS-ссылку и чек-лист оставшихся шагов в @MTProxybot

## Что делает «Настроить безопасность VPS»

1. **Сканирует открытые порты** через `ss -tln`
2. Авто-определяет порт SSH из `/etc/ssh/sshd_config`
3. **Про каждый незнакомый порт спрашивает**: оставить или закрыть
4. **ufw**: открыты SSH-порт, 80, 443, 853 + одобренные пользователем
5. **fail2ban**: бан IP на 1 час после 3 неудачных SSH-попыток
6. **Автообновления**: security-патчи, рестарт в 04:00 если нужно
7. **sysctl**: защита от спуфинга, SYN-flood, MITM, Smurf

## Архитектура

```
Интернет -> :80  -> Caddy (автопродление LE-сертификата, редирект HTTP→HTTPS)
         -> :443 -> Caddy (TLS, страница-заглушка "OK", маскировка под nginx)
         -> :853 -> alexbers (FakeTLS, забирает сертификат с Caddy при старте)
```

## Файлы

|---|---|
| `manage.sh` | Главный скрипт с TUI (чистый ANSI, без зависимостей) |
| `Caddyfile.template` | Шаблон Caddy с плейсхолдером `__DOMAIN__` |

## Безопасность репозитория

**Никогда не коммить**: `config.py`, `.env`, `caddy_data/`, `src/` — всё в `.gitignore`.
