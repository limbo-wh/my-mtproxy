# CLAUDE.md

## Проект

Приватный репозиторий для деплоя MTProto-прокси одной командой.
Архитектура: Caddy (80/443, LE-сертификат) + alexbers/mtprotoproxy (853, FakeTLS).

## Структура

```
deploy.sh              — установщик прокси (интерактивный, запускать на VPS от root)
harden.sh              — защита VPS (ufw, fail2ban, автообновления, sysctl)
Caddyfile.template     — шаблон конфига Caddy (__DOMAIN__)
config.py.template     — шаблон конфига alexbers (__BASE_SECRET__, __DOMAIN__, __AD_TAG__)
docker-compose.yml     — два сервиса: caddy + alexbers
docs/V4.md             — полная инструкция по настройке с нуля
README.md              — краткая инструкция
```

## Секреты — критично

НИКОГДА не коммитить и не выводить в чат:
- config.py (содержит BASE_SECRET и AD_TAG)
- .env (содержит DOMAIN, BASE_SECRET, AD_TAG)
- caddy_data/ (приватный ключ LE-сертификата)
- Реальные IP-адреса серверов, домены, секреты пользователя

Все перечислены в .gitignore. Перед каждым коммитом — проверять git status.

## Рабочий сервер

Не трогать без прямого указания пользователя.
При работе с deploy.sh и harden.sh — только редактировать файлы локально, не подключаться к серверу.
